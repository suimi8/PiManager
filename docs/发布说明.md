# PiManager 发布说明

跨平台 GUI，用于配置、切换、测试和启动官方 Pi Coding Agent（`@earendil-works/pi-coding-agent`）。

当前版本：**v1.7.1**

源码仓库：https://github.com/suimi8/PiManager  
预编译包与 VSIX：https://github.com/suimi8/PiManager/releases/tag/v1.7.1

## v1.7.1 更新

- **全局昼夜模式统一**：管理器页面、已打开弹窗、菜单、帮助 HTML、Qt 文件选择框与 Pi CLI 共用同一个昼夜状态；白天自动同步 Pi CLI `light`，夜间自动同步 `dark`，不再提供独立 CLI 主题设置
- **白天模式黑底修复**：重排顶层窗口样式优先级并补齐完整 Qt Palette，修复 `QDialog`、安装弹窗、消息框和文件选择框在白天模式中混入黑色背景的问题
- **动态主题全面刷新**：主题切换后重新 polish 全部现有控件，刷新导航/按钮/折叠图标、模型状态与延迟语义色、帮助页 HTML，避免保留上一主题的颜色
- **Pi 更新通道修复**：Node.js `22.19+` 使用 npm `latest`；Node.js `20.6–22.18` 使用 `legacy-node20`；低于 `20.6` 时阻止安装并给出明确升级提示
- **Pi 版本识别修复**：只有 `pi -v` 成功退出才解析版本，Pi 启动错误中的 Node.js 版本不会再被误认为已安装的 Pi 版本
- **安装结果强验证**：npm 安装成功后重新执行 Pi 运行时检查；registry 查询失败不再误报“已是最新版”，PATH 中旧 Pi shim 或启动失败会返回可操作错误
- **回归验证扩充**：新增更新通道、registry 失败、安装后验证、全局主题同步、弹窗渲染和模型 Provider/Model 原子切换测试；Python 94 项与 Cursor 扩展 17 项测试通过

## 启动

- Windows 目录版（推荐，启动快）：Release 中的 `PiManager/` 目录
- Windows 单文件版：`PiManager.exe`（首次启动解压稍慢）
- 源码：仓库根目录执行 `python main.py`

## Cursor 扩展

安装 Release 中的 `pi-manager-pi-cursor-*.vsix`。扩展会通过 PiManager helper 解析所选 Provider 的安全密钥。

## 主要功能

1. 自定义 Provider 与模型配置、拉取和批量测试
2. Provider 多 API Key、安全存储和 Pi 子进程运行时注入
3. OS keyring 优先，DPAPI/AES-GCM 当前用户文件库回退
4. 自动迁移旧 `__DPAPI__:` 和明文 API Key 配置
5. 全局代理、健康监控、测试历史和并发测试
6. 配置 ZIP 导入导出，密钥使用 PBKDF2 + AES-256-GCM 加密
7. 会话过滤、重命名、删除和继续
8. Cursor 扩展通过 PiManager helper 安全获取运行时环境
9. Windows、macOS、Linux 终端启动支持

## Provider 密钥

真实 API Key 不写入 `models.json`。配置文件仅保存官方 Pi 支持的环境变量引用：

```text
${PI_MANAGER_PROVIDER_<SLUG>_<HASH>_API_KEY}
```

PiManager 启动官方 Pi 时从安全存储读取真实 Key，并只注入该子进程环境。

同一 Provider 的多把 Key 共用该环境变量引用。明确的 Key 错误会将当前 Key 暂时标为失效并切换下一把；可在 **Provider 管理 → API Keys** 恢复。网络错误和服务端 5xx 不会停用 Key。
